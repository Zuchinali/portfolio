Homelab - Servidor MySQL

Database local para estudos e pesquisas!

Ferramentas:
Linux - Ubuntu, Docker, MySQL
Função:
Database local
Duração:
5 horas

Implementei um servidor local MySQL como parte de um Homelab pessoal, com o propósito central de me desafiar a explorar e dominar o sistema operacional Linux. Este ambiente atua como uma plataforma dedicada para experimentação, onde posso praticar a linguagem SQL, modelagem de dados, gerenciar conjuntos de dados (próprios e fictícios) e o docker. A autonomia desse setup permite aprofundar e fortalecer meus conhecimentos em estatística e análise de dados por meio de estudos práticos e contínuos.

O Desafio

O desafio central era transformar um notebook antigo com Linux Ubuntu em um servidor de banco de dados eficiente para desenvolvimento e análise de dados, utilizando o Docker para garantir isolamento e portabilidade.

O objetivo principal era criar um ambiente de Data Engineering local, onde se pudesse:

  • Hospedar um banco de dados MySQL de forma persistente.
  • Garantir a acessibilidade do servidor a partir de outros computadores da rede (como um PC Windows).
  • Implementar uma interface de gerenciamento simples para o ambiente Docker.

Solução e Metodologia

A solução foi implementada através de quatro fases principais, com foco na estabilidade do sistema operacional, segurança da rede e modularidade do Docker.

  1. Preparação do SO
    Objetivo: Garantir a estabilidade e o acesso remoto.
  2. Instalação do Docker
    Objetivo: Montar a plataforma de virtualização utilizando Docker.
  3. Implantação do SGBD
    Objetivo: Colocar o banco de dados principal em produção (MySQL 8.0 com o Docker).
  4. Gerenciamento e Acesso
    Objetivo: Finalizar a usabilidade do ambiente para poder utilizar remotamente em outro computador e conectar com Power BI, scripts python e Dbeaver.

Dois obstáculos técnicos foram cruciais para o sucesso do projeto:

  1. Problemas de Conectividade
  2. Autenticação do MySQL 8.0

Resultados Chave

O projeto resultou em um ambiente totalmente funcional, validado em dois pilares principais:

  • Servidor Docker Funcional
  • Acesso e Estrutura de Dados
